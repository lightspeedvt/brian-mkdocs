Overview

Note

Please Note this is an unfinished section and a work in Progress. Check back soon for a full section on Interactivity.

"Interactive Lesson" playback in LightSpeed VT is where a number of separate videos are woven together into an interactive experience. It is currently configured via a json formatted text file with specific properties and values which are documented below.

Courseware Management

To create a chapter part with an interactive lesson, first upload your videos to the media library and keep the m3u8 links handy.

preface the chapter part with:

“interactiveLesson|”

Note

We recommend using http://www.jsoneditoronline.org/ to build and modify your .JSON files. It is free, and it makes editing the files quick and easy.

Let's take a closer look at the first section and define what these mean:

{
  "debug": false,
  "videos": {
    "intro": {
      "question": "How much time do you want to spend today?",
      "buttons": [
        {
          "width": 116,
          "height": 47,
          "x": 355,
          "y": 226,
          "value": "5 Minutes",
          "time": [
            0,
            25
          ],
          "video": "5mins"
        },

..............

debug - If this is set to “true” the player will show the button areas visually, which can be helpful for tweaking button placement, etc. (you'll want this to be set to "false" in your final product)

videos - A listing of video configurable objects used in this interactive lesson. Note: the object keys are considered the "id" of the associated video configuration (in our example, "intro" is the first "id").

question - This is the question posed in the interactive video. We track this question as part of the interactive data output, as well as for the mobile version.

buttons - A listing of button configurable objects:

The first Width and Height options are the dimentions of your hotspot/button.

width - Pixel value for the button area's width height - Pixel value for the button area's height

These x/y coordinates are the position on the video where the TOP LEFT corner of the button will lay (See screenshot below for example)

x - Pixel value for the button area's left position y - Pixel value for the button area's top position

value - Text to show when debug is set to true

time - An array in the format of [mins,secs] that tells the player when to activate the button area

video - The id of a video to play once clicked (Note: these are case sensitive and do not allow spaces.)

loopStart - An array in the format of [mins,secs] which tells the video player where to seek to once the video reaches it's end. Used for areas where buttons are presented and an interaction is expected. src - An absolute path to the video to be played
