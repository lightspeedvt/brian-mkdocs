Creating New Locations

To create a new location, navigate to the Super User Dashboard and select "Create New Location" under 102 - Location Management.

This will allow you to create a new location in any System that you currently have Super User Access to.

New Location Information

Step 1:

Select the “System” that you want to create this New Location within.

Step 2:

Fill out the “New Location Information" - starting with which Template Location you want to ‘clone’ this new Location from (more about how Template Locations work is below).

Now you can fill out all of the other information about this Location. The fields with the orange dot indicate a required field. The image below is an example of this screen.

Step 3:

Fill out the Billing Information. First select if this location is “Bill per Location” or “Bill per User.”

And then continue to fill in the rest of the fields. Whether you select “By Location” or “By User” all of the fields apply.

Setting
Gross Set Up Fee If you charged a one time set up fee, put in the dollar amount here.
Gross License Fee This is the price is was sold for.
License Frequency This is how often the license fee is charged; in month(s). A monthly charge would be “1” and annual charge would be “12” (12 months), or leave it blank if it is a charge that is incurred once, and never again.
Begin Bill Date This is inserted for you with the “creation date” of this New Location. If you need this changed, please specify this in the “Billing Notes” filed at the bottom of this section. This will also be considered the Contract Start Date.
Contract End Date This is an optional field where you can indicate when this contract expires.
Grace Period This only applies when the “Auto Expire” Check box is indeed checked. The Grace Period allows you to give the Location a grace period to renew by. If you choose to Auto Expire a Location, it will expire on the Grace Period Date. Alternatively, if no Grace Period Date is entered, the Location will expire on the Contract End Date.

Note

If you intend for this Location to renew in a ‘month to month’ format after the Contract End date, then do not check the “Auto Expire” check box, so it will not get automatically deactivated on that date and it will roll over to a ‘month to month’ status.

Setting
Billing Notes This is an open text field for you to note anything you would like the LSVT Accounting Dept. to know. These notes are only seen by you and the LSVT Accounting Dept.- they are not seen on the front end by your customers.
Contract This is where you will upload the contract for this New Location to be stored.

Step 4:

Location Notes - this is an optional open field that you can utilize to indicate any helpful ‘notes’ about this Location that you and your customers can see on the front end to better describe what the Location is being used for. You can also update this text in the “Location Settings” at any time.

When you are ready, click the “Create Location” button and this Location is ready to use right away.

Note

This newly created Location is ready to use right away, but it will also be sent to the LSVT Accounting Dept. in a “Pending Approval” status for review. If the LSVT Accounting team has any questions about it, they will contact you directly prior to placing it in an “Approved” status.

Template Locations

When new Locations are created, we follow this process of cloning them from an existing “Template” Location so that the basic settings are all in place, and make it easier and faster for you as an Administrator to create new Locations. It also helps you ensure that it is created correctly. Below are a list of settings that are cloned in this process.

  • The Theme - this process will place the new Location in the same System and use the same Theme as the Template Location.
  • Courseware - all of the courses that are active in the Template Location will be the exact set that are active for this New Location.
  • Super User List - all of the Super Users that have access to the Template Location will now also have access to this newly created Location.
  • Main Menu messages - the Main Menu welcome message video, the Main Menu bulletin board message and the left and right images and messages will all be used.

Other Location Management Settings

Various Super User B Privileges will grant you access to more available options. These will typically be available to your top Admin.

Course Settings & Copy Courses (With Super User B Privilege )

On the Course Settings tab, you can activate or deactivate content at the location, or export a list of content available at this location.

On the Copy Courses tab, You can use this feature to copy the course settings from a different location onto the one you are currently in.

Billing, Assigned System & Recent Activity (With Super User B Privilege)

The Billing tab will show you a read-only output of the billing setup for this location. You can also add an expiration date for the contract or add billing notes about the location.

The Recent Activity tab will show you recent changes made to the location settings.

The Assigned to System dropdown will allow you to move this location into another System (You will typically never need this.)

Location Activation (Super User B Privilege)

With access to the Super User B Privilege for Location Activation, you will be able to activate or deactivate a location. This action is tracked in the "Recent Activity" tab as well.

User Cap - Max # of Active Users (Super User B Privilege)

This privilege will allow you to modify the maximum number of active users allowed at this location.

Ghost Users (Super User B Privilege)

Ghost Users is a feature that will hide users at your location so that they no longer count against the Max # of Users limit.

As an example, say you require "20 regular users and 1 admin" for a new location, but you do not want the admin to be counted toward the user limit - this feature will allow you to do that.

Access to Ghost Users is only available at locations where the accounting format is set to "By Location - Application Model (Flat Fee)" OR "By Location - Generation Model (Rev Share)."

To activate Ghost Users, select the "yes" toggle and then pick which access level should be a ghost user.
