Overview

This article is a “quick” overview on how the whole “Courseware Management” area works. The other subsequent articles will give you deeper descriptions and literal “step-by-step” walkthroughs of every feature and function, but if you read and understand this overview first, you’ll be able to follow along much more clearly.

When your VT Account is created, it comes along with a Content Container and this is where all of your content is “contained” and it is what feeds all of the Systems, Private Labeled Themes, Locations and Users underneath it. This diagram below walks you through the 5 basic steps of getting content created from scratch - to ultimately available for your end users to see.

Note

“Courses” are the ON/OFF building block across the whole application. This will become clearer as soon as you understand how they interact with “Content Roles” - “Locations” and “Users.”

Step 1: Upload your videos into the Video Library
The Video Library will let you upload any format of “source” video into it, and it will do all of the encoding for you to all of the different formats that we need. When a video is in the Video Library, you can reuse that video asset anywhere in the application that you want to.

Step 2: Assemble your videos into Courses
Once you have one or more videos in the Video Library, you can create/edit Courses and Chapters and add the videos to them as needed. As you add Chapters, you will also see that you can upload videos right “in-line” while you are editing Chapters as well (vs. having to go add them to the Video Library first). These videos will also be added to the Video Library for you automatically.

Step 3: Add your courses to Content Roles
So far we have talked about “Videos” and “Courses” ... and now on to “Content Roles” ... all of these things are part of your “Content Container” as illustrated in the top large gray box that represents your entire “Content Container.” Content Roles allow you to organize your Courses as you wish. These Content Roles are represented by the yellow rectangles in the diagram above, and they have “Courses” within them. You can have “1” Course in a Content Role, or “1,000,000,000.” You can also put the same Course into many different Content Roles if needed.

For example, let’s say you have these Content Roles:
Sales Strategies - (this has courses on sales training) AND (the New Hire Orientation course)
Compliance - (this has courses on mandatory compliance training) AND (the New Hire Orientation course)
Front Desk Training - (this has courses on how to run the front desk) AND (the New Hire Orientation course)

Step 4: Make Courses “Active” at each desired Location
All of the Courses that are Active in your Content Container may be turned ON or OFF at each/any Location. Once a Course is Active at a Location, it may very well be inside a Content Role that a User has, and then a User at a Location will see that Course.

Step 5: Each User can be set to as many Content Roles as desired
The list of available Content Roles for each user, is the list that is established from your Content Container. Once a User is set to a Content Role, they will see the list of Courses that are Active in that Content Role - as long as each of those Courses are “Active” at their Location.

In the diagram above, there is a dotted line that represents the “Location stuff.” For each Location there is a Course List, and we show two of the Courses as being “Inactive” and they are grayed out. You will then see in the Content Roles for the User (the yellow rectangles) that there are the two grayed out Courses - one in each Content Role. Here is an example where that flexibility can be really helpful:

Let’s say you have a Content Role called “Compliance” and you have two different courses in it, one is “Compliance Laws” and a separate, but similar course is “Compliance Laws for California” (for some reason California seems to have way more laws about Compliance than anywhere else).

You can put both versions of your Course in the Content Role called “Compliance” - and then when Managers create users, they will simply assign them the Content Role of “Compliance.” However, you can Activate the “California” Compliance Course only at the Locations in California, and the regular Compliance Course everywhere else.

This flexibility may be more than what you need, but it is all there to accommodate any type of content activation needs you may have.

Note

There is a way to make some Content Roles “Premium” so that they are not visible at the Location level to be set to users, but you as the Top Admin can see them. This gives you even more flexibility for when you want to start offering premium content to Locations as an up-sell item that can be purchased. We will go into this more later in the “in-depth” area of these Help Docs.

So, that is the “quick” overview of how Courseware Management works. You can find more details on each area below.

Categories

This section will help you create a new Category. A category is made up of one or more courses, a text description, an image thumbnail (picture or image representing the content) and if applicable, a rollover image thumbnail (an additional image that will appear when a user hovers over the tile with their mouse).

Example of the Training Center before you hover over a category tile with your cursor
Example of a cursor rolling over the center Category

After you have selected your content container, you can select Category List, where you can either Search for an existing category, or Create a new category.

Create or Edit a Category - Basic Info

  1. Active Yes/No - Default: Yes
    • Select whether this category should be active or not. (Note: It is safe to always select "Yes" here - this content will not be on display for anyone until you complete the very last step of the activation process.)
  2. Rank - Default: 1, Required Field
    • The Rank is the order in which this category will display in the training center. The lower the rank (1, 2, 3, etc.) the higher the category will be listed in the system. Rank 1 = the very first category. Rank 999 = the very last category. If two or more categories are set to the same Rank, then the system will sort them alphabetically.
  3. Category Type - Standard or List of Links
    • A category can be "Standard" or it can be a "List of Links". A "List of Links" means you can use a category to point to additional categories, OR you can use the category to link out to another website, link to a file to download, etc. You may use the "List of Links" Category Type to add additional organization layers to your Training Center. "Standard" categories, when clicked, will simply load the courses that are assigned to that category.
  4. Name - Required Field
    • Enter the name of the category here.
  5. Thumbnail
    • Here you can enter the image path to the category image, or choose to upload one from your local device.
  6. Thumbnail Roll Over (Optional) - This can be left blank
    • Here you can enter the path to the category rollover image, or choose to upload one from your local device. This is similar to the Thumbnail above.
  7. Add Sheen to Thumbnail - Default: Yes
    • The "Yes" setting will automatically add rounded edges and a highlight effect to the rollover. If you set this to "No", it will assume you are adding the rollover and rounded corners through other means (Custom CSS, GIF, etc.)

Example:

Add Sheen = Yes Add Sheen = No Charlie Sheen = Winning

Warning - File Size Best Practices

LSVT does not compress or resize your images. Every time one of your users load the Training Center they will be downloading all of the visible artwork on the screen. This means you should keep the file size low, and use the correct dimentions, when uploading these thumbnails. For example, if you add a 50mb image to one of your Category tiles, the user is going to have to download that every time.

Category - Additional Information

  1. Description - Can be left blank
    • Enter the description of the category.
  2. Internal Description - Can be left blank
    • Enter the internal description. This is an Internal view for you only. This can be used to add additional important details about the course, or even to log recent changes.

When you have entered all of your information, click "Save", and move on to the next section.

Courses

A Course is made up of one or more Chapters, a text description, an image thumbnail (picture or image representing the content) and a Course Preview Trailer (video).

After you have selected your content container, you will be on the Course List screen, where you can either Search for an existing course, or Create a new course.

Create or Edit a Course - Basic Info

  1. Active Yes/No - Default: Yes
    • Select whether this course should be active or not. (Note: It is safe to always select "Yes" here - this content will not be on display for anyone until you complete the very last step of this process.)
  2. Rank - Default: 1
    • The Rank is the order in which this course will display in the training center. The lower the rank (1, 2, 3, etc.) the higher the course will be listed in the system. Rank 1 = the very first course. Rank 999 = the very last course. If two or more courses are set to the same Rank, then the system will sort them alphabetically.
  3. Course Type - Default: Standard
    • A "Linked Course (Banner)" indicates that this course will be used to link to another course, rather than linking to a set of chapters as a Standard course would.
  4. Name - Default: Required Field
    • Enter the name of the course here.
  5. Category - Default: Required Field
    • Select which category this course should go in. (Note: Currently a course can only be in 1 Category at a time.)
  6. Thumbnail
    • Enter the path to the course image. This will be the web URL to where the image is hosted on your server. Or, you can click on "Choose File" to upload a thumbnail from your local device.
  7. Preview Video - This can be left blank.
    • Enter the path to the course video preview (If you do not have a course preview video, just leave this blank.)
  8. Enable Notes - Default: Yes
    • This will toggle on and off the notes feature for the entire course. Additionally you can control the notes feature at the System level, and at the Chapter level.

Course - Additional Information

  1. Course Description - This can be left blank.
    • Enter the description of the course.
  2. Course Internal Description - This can be left blank.
    • Enter the internal description. This is an Internal view for you only. This can be used to add additional important details about the course, or even to log recent changes.
  3. Author - This can be left blank.
    • Enter the name of the author of this material.
  4. Duration - This can be left blank.
    • This field is for the duration, in minutes, of the course. (Note: Currently is field is not output in the Training Center, we will be including it in a future release.)

Course - Display Settings

  1. Hide this Course from Reports - Default: No
    • This will show/hide the course in reporting sections.
  2. Hide this Course from Search - Default: No
    • This will toggle the course from being searchable.
  3. Complete Chapters in Order - Default: No
    • Setting this toggle to "Yes" will force the user to complete the chapters in order.

Course - Keywords

  1. Keywords - This can be left blank.
    • Add any keywords associated with this course. Type in your desired Keyword and then hit enter. Enter as many keywords as you desire.

Course - Extended Info

  1. Extended Info - This can be left blank.
    • In this area you can add additional data to the Course Menu, if needed, such as additional resources for the user to download, or adding special instructions for the end user.

Course - VT2GO Settings

VT2GO settings are covered HERE.

Course - Content Role Assignments

  1. Content Role Assignment
    • Select each content role that this course should appear in. Note: This still does not mean the content is active. If you recall from the diagram at the top of this section, the final step will be to activate this course for the Location. Note 2: LightSpeed VT will create you an "Admin - All Content" role initially for testing and review purposes, however if you need to add additional roles, you may do so now, or after your Courseware is created. To add more Content Roles now, click on "Content Containers" in the top navigation bar now. We cover Content Containers and Roles in the Content Containers section of these Help Docs.

When you have entered all of your information, click "Save", and move on to the next section.

Chapters

Once a user clicks into a Course, they may select a Chapter of their choice, unless you changed the toggle to require them to take the chapters in order.

A Course may have as many Chapters as needed. Each Chapter may consist of multiple Chapter parts. Each Chapter part consists of either an M3U8 Playlist (for Streaming content) or an MP4 file (for Progressive).

  • A Chapter may consist of:
    • Training parts and a Chapter test.
    • Training parts and no test.
    • No training parts and only a test.

By default, a chapter with a test is followed by a "Recap". A "Recap" will play if the user fails to pass the test. This is optional, howevever it is recommended you include these. (In some scenarios, if you have a 10 minute chapter, and the user fails the test, it is a better user experience to have the user review a brief recap, 1-2 minutes, rather than go through the entire 10 minute video again)

Chapter creation is unique from Categories and Courses in which you can only "Create New Chapter" from an existing course. So you'll see after you save your course from the previous section that there is now a "Create New Chapter" button at the top.

Click on that when you're ready to create your chapter.

Create New Chapter - Basic Info

On the Create New Chapter menu, we see the following fields (click on the image to open it in a new window):

Create your chapter by filling out these fields:

  1. Active
    • If this is a standard chapter, you must add at least 1 test question in order for you to receive the option to activate it.
  2. Rank - Default: 1
    • The Rank is the order in which this chapter will display. The lower the rank (1, 2, 3, etc.) the higher the chapter will be listed. Rank 1 = the very first chapter. Rank 999 = the very last chapter.
  3. Chapter Name
    • Enter the name of the chapter here.
  4. Chapter Type:
    1. Standard (Video and Test)
    2. Test Only
    3. Video Only (with Auto Advance)
    4. Video Only (with Summary)

Let's take a closer look at these chapter types.

Adding Videos to your Video Library

1. Standard (Video and Test)

In a Standard chapter type, the chapter must have at least 1 video part and at least 1 test question.

After selecting 'Standard', you will need to click on 'Add Chapter Part' and either upload a new video to your Video Library, or copy and paste the path of an existing video in your Video Library into the chapter part field.

Summary Info

  1. Test Instructions / Setup (Seen before test)
    • Enter the test instruction text the user will see before starting the test. The default language we use here is:

      This chapter test is a quick check of your knowledge so far. As you select your answers, each question is timed and all results are recorded on your “Report Card.” Click START below when you are ready.

  2. Test Summary (Passed Test) (Seen upon test completion)
    • Enter the test summary for when the user passes the test. The default text is already populated if you do not need to change it.
  3. Test Summary (Failed Test) (Seen upon test completion)
    • Enter the test summary for when the user fails the test. The default text is already populated if you do not need to change it.

Test Settings

  1. Number of Test Questions - Default: 1
    • Enter the number of test questions that will be pulled for this chapter. If you enter a number here smaller than the total number of test questions active, then it will randomly select that number from that "pool" of available questions. For example - if there are 10 active test questions for this chapter, and the number of test questions is set to 5, each time a user takes the test it will randomly serve up 5 of the 10 total questions.
    • Note: You will not be able to activate this chapter until this number matches the number of test questions added.

  2. Test Pass Percentage - Default: 100

    • Enter the minimum test percentage a user must receive in order to pass the test and move on to the next chapter.

  3. Test Type - Default: Standard

    • Your options here are: Standard, Survey, Only Test, No Test. More on testing mechanisms can be found here.
  4. Pull Questions in Order By Rank - Default: No
    • This will pull all of the test questions in order by rank, and no longer serve them up randomly.
  5. Fail Test on Incorrect - Default: No
    • Setting this to "Yes" will cause the user to fail the test on ANY missed question - rather than allowing them to complete the entire test first.
  6. Answer Notifications - Default: No
    • Setting this to "Yes" will allow you to enter an email address per answer which will notify the recipient of that user's answer. This is often used with the Survey test type.
  7. Show Test Stats - Default: Yes
    • This toggle will show or hide the test stats before and after the test. This is often used with the Survey test types.
  8. Show Test Summary Nav Buttons - Default: Yes
    • This toggle will show or hide the navigation buttons after the test.
  9. Enable Notes - Default: Yes
    • This will toggle on and off the notes feature for this specific chapter. Additionally you can control the notes feature at the Group level, and at the Course level. This should always be set to "Yes" unless indicated otherwise.
  10. Social - Leave blank for now.
    • You can enter in social information per chapter.
  11. Chapter Part Info - Default: 1
    • You must enter at least "1" chapter part here.

Once you have filled out each field, hit Save. After the course is created there will be some additional fields to complete before moving on.

Create New Chapter - Part 2

  1. Show Captions - Leave blank for now.
    • Selecting this button will show the optional Caption fields.
  2. Video Path, Mobile Path, Asset Path
    • Video Path: This is the path to the chapter part media (MP4, PDF, etc.)
      • For a default video for a "LaFR" we use: http://videos.lightspeedvt.com/video_files/coming_soon.mp4 - this is a path to a placeholder "Coming Soon" video. This is used so we can show our clients a completed system without actually having any of their content ready.
    • Mobile Path: This is the path to the mobile encoded version of the above video path. Leave Blank for now.
    • Asset Path: This is a legacy field, it is for content that still utilizes flash side assets that lay on top of the video parts, before they were edited into the video itself. Most of the time this can remain blank. Leave blank for now.
  3. Rehash/Recap - Video Path
    • A recap HTML file is linked here by default. If the user fails the test then the user will be prompted to take a recap before taking the test again. This HTML file will let the user know they have 1 more chance to pass the test or else they'll have to watch the chapter part over again.
  4. Add Question - Be sure to hit "Save" before you hit "Add Question" - as you will lose any modifications you have made to the chapter if you do so.
    • This is where you can add test questions to this chapter. We'll cover that in the next section.

Once all of the chapter information has been added, you're ready to move on to the next section - adding test questions.

Test Questions

Create new Test Question

After your chapter has been saved, click on "Add Question" and follow the steps below.

  1. Question
    • This is where you enter the question text. For this exercise, we'll enter a placeholder question. Enter this text into the Question field: Did you understand everything in this chapter?
  2. Question Active - Default: Yes
    • This is a simple toggle - this will default to yes.
  3. Question Rank
    • This field will increase in rank for each test question you add. It is only used when the chapter toggle "Pull Questions in Order by Rank" is set to "Yes".
  4. Answer A) & B)
    • This field is for the first possible test question answer. For this exercise enter this as the answer: Yes
    • In the Answer B) field, enter: No
  5. Show Correct / Inc. - Default: Checked
    • These checkboxes will be selected by default. If this is unchecked, after a user answers a test question, the "Correct" and "Incorrect" dialogue pop-ups will not be active.
  6. Response Action - Default: None
    • This drop-down box is where you can setup an answer response. You can either select a video response or a text response. If a video response is selected (FLV), then you will enter a path to that video in the Response Info field. If Text is selected, then you will enter your text answer response in the Response Info field.
  7. Response Info - Leave blank for now.
    • This field will be filled out only if the Response Action field is active.
  8. Points - Leave blank for now.
    • You can assign a point value to each different answer. This feature was designed to eventually lead to a "Leaderboard" where users can see how they rank against other users. Currently this feature is disabled, but the field was left here for when the development team eventually circles back to it.
  9. Correct Answer
    • Here you can select "Any / All Correct", or any individual answer. If you hold Ctrl/Command you can select more than one answer. For this exercise select answer "A".
  10. Question Timer - Default: 60
    • Here you can set, in miliseconds, how long the question timer should be. We default to 60 seconds.

Once all of the fields are complete, hit "Save" and then click on the chapter name at the top of the screen to return to the chapter menu. From there you can now activate the chapter.

And that is it. You should have now successfully created a Category, Course, Chapter and Test Question.
